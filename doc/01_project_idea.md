# Project Idea: RAG-ассистент по фильмам и сериалам

## Проблема
Зрителям сложно быстро найти фильм или сериал под конкретное настроение, описание или набор условий (например: «детектив 2022 года, но не слишком страшный»).
Поиск по ключевым словам в базах данных (IMDB, Kinopoisk) не учитывает семантику и контекст — выдает либо слишком много вариантов, либо нерелевантные.

## Цель проекта
Создать RAG-систему, которая:
- По описанию или вопросу на естественном языке находит релевантные фильмы/сериалы.
- Формирует ответ с названием, кратким описанием и источником (ссылка на датасет или внешний сайт).
- Работает локально и показывает полный pipeline от данных до ответа.

## Для кого проект
Студенты, изучающие RAG, а также обычные зрители, которые хотят подбирать кино по смыслу, а не по ключевым словам.

## Какие данные используете
- **Источник**: Kaggle — `IMDB Movies Dataset` или `TMDB 5000 Movie Dataset` (выбираю TMDB, т.к. там есть описания, жанры, ключевые слова, рейтинги)
- **Ссылка на данные**: (добавлю позже, скачаю с Kaggle)
- **Примерное количество записей**: от 5000 фильмов (для оценки "отлично" — 1000+ записей или чанков)
- **Формат данных**: CSV / JSON (поля: title, overview, genres, release_date, vote_average)
- **О чём данные**: названия фильмов, их описания (overview), жанры, дата выхода, рейтинги

## Критерий успеха
1. RAG отвечает на 3 вопроса по данным, например:
   - «Какой научно-фантастический фильм про путешествия во времени вышел после 2015 года?»
   - «Посоветуй комедию с высоким рейтингом для вечера»
   - «Что за фильм, где герой теряет память и расследует убийство?»
2. Корректно отказывается на 1 вопрос не по теме:
   - «Как приготовить пиццу?» → «Извините, я знаю только про фильмы и сериалы»
3. Streamlit UI показывает ответ + источники (название фильма, ссылка на TMDB или строку в датасете)
4. Код запускается по инструкции из README на чистой машине
